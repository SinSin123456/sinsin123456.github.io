Student Management System

The system that supports two primary roles: Admin and Student User. It was developed to practice advanced Python concepts, including file handling, authentication, error handling, and data operations.
View Project Requirements

This is a role-based Student Management System designed using Python + SQLite, with the database securely stored in the cloud (Google Drive). It aims to create system that manages students operations efficiently, with secure, role-based access.

Key Features:
Admin Feature

  • Admin login system with secure credentials.
  • Create and manage user accounts (students).
  • User login attempt limited to 3 tries (auto-lock on failure).
  • Unlock user accounts.
  • Perform full CRUD operations on users

Student Feature

  • Login with a 3-attempt limit before locking.
  • View personal grades
  • Add new subjects.
  • Drop enrolled subjects.
  • Pay fees for each subject
  • View a detailed report of enrolled subjects, payments, and academic performance

Data Storage

  • The database is stored locally using SQLite.
  • Periodic sync to Google Drive for cloud backup and accessibility
